# 组织稳定性评估标准

OrgRadar 的组织稳定性评分用于判断，最近一段时间内，组织变化是否处于正常、可解释、可接受的范围内。

评分不针对所有变化。新增、调岗、组织调整都可能是正常经营行为。评分重点关注变化是否异常、是否集中、是否影响关键人员、是否体现出组织收缩或局部流失。

## 统计窗口

默认统计最近 30 天。

## 状态分级

| 分数 | 状态 | 含义 |
| --- | --- | --- |
| 90-100 | 稳定 | 组织变化处于正常范围 |
| 75-89 | 关注 | 出现局部波动，需要观察 |
| 60-74 | 波动 | 变化较明显，建议管理层关注 |
| 0-59 | 风险 | 出现明显组织风险或异常变化 |

## 评分结构

满分 100 分，按以下维度扣分：

| 维度 | 最高扣分 | 说明 |
| --- | ---: | --- |
| 人员流失影响 | 35 | 按近 30 天离职率计算，不以绝对人数为准 |
| 净减少影响 | 15 | 入职可以抵消离职带来的净减少影响 |
| 部门集中影响 | 25 | 判断离职是否集中在少数部门 |
| 组织调整影响 | 15 | 部门变化、职位变化是否频繁 |
| 关键人员影响 | 10 | 管理员、负责人、带下属人员变化 |

计算公式：

```text
组织稳定性评分 = 100
- 人员流失影响
- 净减少影响
- 部门集中影响
- 组织调整影响
- 关键人员影响
```

## 人员流失影响

```text
离职率 = 近 30 天离职人数 / 当前在职人数
```

| 近 30 天离职率 | 扣分 |
| --- | ---: |
| 0% | 0 |
| <= 0.5% | 4 |
| <= 1% | 8 |
| <= 2% | 15 |
| <= 3% | 22 |
| <= 5% | 30 |
| > 5% | 35 |

## 净减少影响

```text
净减少人数 = max(0, 离职人数 - 新增人数)
净减少率 = 净减少人数 / 当前在职人数
```

| 近 30 天净减少率 | 扣分 |
| --- | ---: |
| 0% | 0 |
| <= 0.5% | 3 |
| <= 1% | 6 |
| <= 2% | 10 |
| > 2% | 15 |

入职不直接加分，但会抵消净减少扣分。

## 部门集中影响

部门集中影响用于判断是否存在局部部门流失。

```text
部门离职率 = 部门近 30 天离职人数 / 近似部门基数
近似部门基数 = 当前部门人数 + 近 30 天离职人数
```

| 条件 | 扣分 |
| --- | ---: |
| 无部门集中流失 | 0 |
| 部门人数少于 5 人，且离职人数 >= 2 | 4 |
| 部门离职人数 >= 2 且部门离职率 >= 5% | 10 |
| 部门离职人数 >= 3 且部门离职率 >= 8% | 18 |
| 部门离职人数 >= 5 或部门离职率 >= 12% | 25 |

小部门的一人变化只作为提醒，不直接重扣分，避免小样本导致评分失真。

## 组织调整影响

```text
组织调整次数 = 部门变化次数 + 职位变化次数
组织调整率 = 组织调整次数 / 当前在职人数
```

| 近 30 天调整率 | 扣分 |
| --- | ---: |
| 0% | 0 |
| <= 1% | 3 |
| <= 2% | 6 |
| <= 5% | 10 |
| > 5% | 15 |

## 关键人员影响

关键人员优先使用现有通讯录字段识别：

- `is_admin`
- `has_subordinate`

| 条件 | 扣分 |
| --- | ---: |
| 无关键人员变化 | 0 |
| 关键人员职位变化 | 3 |
| 关键人员部门变化 | 5 |
| 关键人员离职 | 10 |

同一统计窗口内，关键人员影响最多扣 10 分。

## 解释口径

页面不只展示分数，还会展示主要原因，例如：

```text
近 30 天离职 3 人，整体离职率 1.1%
市场营销中心出现 2 人离职，存在局部流失
近 30 天净减少 1 人，入职已用于抵消净减少影响
```

异常提醒用于解释和观察，不再直接作为额外扣分项，避免同一件事被重复扣分。

## 设计原则

- 按比例判断，避免大公司和小公司使用同一绝对人数阈值导致失真。
- 入职不直接加分，防止用招聘增长掩盖组织流动。
- 局部集中流失比平均流失更值得关注。
- 关键人员变化单独表达，但设置上限，避免过度放大。
- 提醒和评分分离：提醒解释风险，评分衡量稳定性。
